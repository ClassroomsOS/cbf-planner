# Claude Code Session Checklist

**IMPORTANTE:** Claude debe leer este archivo al inicio y final de cada sesión.

## Al FINALIZAR cada sesión

Antes de terminar la conversación, Claude DEBE ejecutar estos pasos en orden:

### 1. 🚨 VERIFICAR COMMITS (OBLIGATORIO)
```bash
./.claude/session-end-check.sh
```
**Si el script falla (exit code 1) → NO PUEDES CONTINUAR hasta commitear**

Si hay cambios pendientes:
```bash
./.claude/auto-commit.sh "feat(scope): descripción"
```

### 2. 📋 CHECKLIST DE DOCUMENTACIÓN

Preguntar al usuario:

```
🔍 Checklist de fin de sesión:

1. ¿Implementamos features nuevas que NO están en CLAUDE.md?
   - [ ] Nuevas funciones de AI
   - [ ] Cambios en arquitectura/estructura
   - [ ] Nuevos componentes o páginas
   - [ ] Cambios en modelo de datos
   
2. ¿Hay código que debe documentarse?
   - [ ] Nuevas convenciones
   - [ ] Cambios en flujos existentes
   - [ ] Decisiones arquitectónicas importantes

3. ¿Actualizamos CLAUDE.md con estos cambios?
```

Si el usuario responde SÍ a cualquier punto → **ACTUALIZAR CLAUDE.md + COMMITEAR**

## Al INICIAR cada sesión

Claude debe verificar:

1. **¿Hay commits recientes sin documentar?**
   ```bash
   git log --oneline --since="24 hours ago"
   ```
   Si hay commits con mensajes como "feat:", "refactor:", "fix:" → preguntar si CLAUDE.md fue actualizado

2. **¿Hay archivos modificados recientemente que puedan necesitar documentación?**
   ```bash
   git diff HEAD~5..HEAD --name-only | grep -E "AIAssistant|Context|Page.jsx"
   ```

---

## Rationale

**Problema:** El usuario trabaja en múltiples sesiones de Claude Code concurrentemente. Los cambios se commitean pero CLAUDE.md no se actualiza, causando:
- Features implementadas pero no documentadas
- Trabajo duplicado en sesiones diferentes
- Pérdida de contexto entre sesiones

**Solución:** Checklist automático al inicio/fin de cada sesión para forzar sincronización de CLAUDE.md con el código real.
